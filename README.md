# NCAA Baseball Camps 2026 Directory

The definitive, searchable guide to 2026 NCAA Division I and Division II baseball camps — 559 programs.

**Live Site**: [bmwseals.com/Baseball_Camps_2026/](https://bmwseals.com/Baseball_Camps_2026/)

## Features

- **559 Programs**: Complete DI + DII coverage with real camp data or clear "TBA" indicators.
- **Premium UI**: Dark theme with Inter/Outfit typography, smooth card animations, and responsive layout.
- **Multi-Dimensional Filtering**: Division, conference, price range, latest dates, latest updates, and community verification.
- **Dynamic Rendering**: Production HTML (`index.html`, ~64KB shell with runtime JSON fetch). Static backup generator available at `generate_html_backup.js` (~5.3MB inline, kept as fallback).
- **V12.6 Ultra-Fidelity Extraction**: Intelligent Puppeteer-based scraper with mascot-based search, portal bridge logic (Ryzer, PlayNSports), and automated session detection.
- **Watchdog Stability**: Self-healing process monitor prevents scraper hang or crash.
- **Community Verification**: Users can verify individual schools; counts sync live from server.

## Quick Start

```bash
npm install                     # install dependencies
npm run generate:html           # rebuild index.html from camps_data.json (dynamic, ~64KB shell)
npm run generate:dev            # rebuild index_dev.html (same dynamic approach, dev deploy)
npm run generate:backup         # rebuild index_1.html (static 5MB backup, fallback)
npm run audit                   # run quality audit
npm run deploy                  # deploy production
npm run deploy:dev              # deploy staging (/Baseball_Camps_2026_dev/)
npm run full-update             # sync verifications → audit → generate → deploy
```

## Architecture

1. `smart_extract.js` → V12.6 Puppeteer engine writes data to JSON
2. `generate_html.js` reads JSON and produces `index.html`
3. `auto_verify.js` validates URLs; `quality_audit.js` scores data completeness
4. `deploy.js` uploads via FTP (credentials in `.credentials/`, gitignored)

## Project Structure

- `camps_data.json` — Master database (559 programs)
- `index.html` — Production directory (generated, dynamic ~64KB shell)
- `index_dev.html` — Dev staging (generated, same dynamic approach)
- `index_1.html` — Static backup (generated by generate_html_backup.js, ~5MB inline bundle)
- `smart_extract.js` — Extraction orchestrator
- `generate_html.js` — Dynamic renderer → `index.html` (production)
- `generate_html_dev.js` — Dynamic renderer → `index_dev.html` (dev deploy)
- `generate_html_backup.js` — Static bundler → `index_1.html` (5MB fallback)
- `deploy.js` / `deploy_dev.js` — FTP upload scripts
- `blacklist.json` — Centralized junk third-party domain filter
- `src/utils/` — Extraction engine, config, mascot lookup, Word export
- `src/tests/` — Unit tests and verification scripts
- `src/archives/` — Deprecated/historical scripts
- `src/debug/` — Debugging sandboxes
